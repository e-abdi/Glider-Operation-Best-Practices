    Slocum Deployment Checklist¶

    Print this page

    Deployment mission:  ________________________________   Glider S/N:  _____________

    Engineer:  _________________________________________   Date:  ________________

    1. Pre-Deployment¶

    • Archive deployment on SFMC
    • Cache files
    • Test remote comms  Cell / Iridium Go / Iridium handset
    • Remote team confirms local testing done
    • Remote team relays glider turnaround plan and ETA
    • IMS movements confirmed

    MicroRider (if fitted)

    • MR — Check probe capacitance/resistance with multimeter/Gigaohm-meter
    • MR — Check SETUP.CFG (serial numbers, cal constants, file names: D_*.P)
    • MR — Bench test with test probes; check P-file creation and logfile for errors  odas5ir -f setup.cfg -c all
    • MR — Check/set date and time
    • MR — (optional) Bench test with mission probes; check P-file and logfile; reinstall test probes after
    • MR — Check probe ports (greased O-ring, ferrule OK)
    • MR — Check all screws, bolts, and connectors are firm/tight

    Final pre-deployment

    • (optional) Run a simulation; check all files and data
    • Download all relevant glider/sensor files and build the final Expedition folder; send to office/SharePoint  *.ma, *.mi, config files (science and flight), MicroRider SETUP.CFG

    2. On Deck¶

    • Note deployment time  Ensure remote team has unplugged Freewave
    • Stop initial.mi  Ctrl-C × 2
    • Increase time in GliderDOS  put u_max_time_in_gliderdos 3600
    • (optional) Pull engineering config files  zs config/autoexec.mi   zs config/sbdlist.dat
    • (optional) Pull science config files  szs /config/proglets.dat   szs config/tbdlist.dat
    • Confirm vanilla mission and .ma files are present  (optionally type send off or send on)
    • Send new goto and any new files  dockzr *.*

    MicroRider (if fitted)

    • MR — Install mission probes  confirm probes installed by remote team
    • MR — On-deck calibration; fill ChanStats checklist; check for broken probes  u4stalk, then odas5ir -f setup.cfg -c all

    Status checks

    • Run status.mi  run status.mi
    • Confirm status.mi completes normally — no aborts, or any aborts understood and resolved
    • Confirm GPS location; send SBD/TBD; check age of GPS fix
    • Plot engineering and science data  send *.sbd *.tbd
    • Confirm engineering data in DataVisualizer
    • Confirm science data in DataVisualizer  battery · vacuum · time · GPS position · leak detectors
    • Confirm devices show no unusually high number of warnings or oddities
    • Communicate to remote team: glider is OK / not OK to deploy  From this point the pilot team is responsible for control of the glider

    3. In Water¶

    • Remote team confirms glider is deployed and is on/off buoy
    • Run status.mi  run status.mi
    • Confirm surface dialog engineering data  battery · vacuum · time · GPS position · leak detectors
    • Confirm no aborts
    • Zero ocean pressure  zero_ocean_pressure

    MicroRider (if fitted)

    • MR — Surface calibration; fill ChanStats checklist; check for broken probes  u4stalk using ctrl-F, then odas5ir -f setup.cfg -c all
    • MR — Send calibration data to Rockland support  copy calibration text from terminal

    Initial dive sequence

    • Run ini0.mi (on buoy preferred)  run ini0.mi
    • Send MDB  send *.mbd -num=1
    • Send all SBD and TBD (should be ini0 and status only)  send *.sbd *.tbd
    • Exit reset and catch glider on connection  exit reset
    • Confirm ballast — dive/climb profile should be a clean V-shape  if not, use drive calculator
    • If not V: inform field crew of weights to add/remove and repeat ini0
    • If ballast OK: notify crew to remove buoy
    • Once glider is free: sequence mission  Sequence mission.mi(5) (or expedition-specific mission name)
    • Confirm first vanilla mission dive depth is 30 m
    • Confirm script is running  SFMC-g3s-ma-first-num=3.xml

    MicroRider (if fitted — after first dive)

    • MR — Surface calibration after first dive; fill ChanStats checklist
    • MR — Check MicroRider memory: confirm increasing data files and growing log size  dir \DATA\D*.P   dir \DATA\logfile.txt
    • MR — Mid-profile channel analysis from .tbd data; fill ChanStats checklist (mid-profile)  reference: TN_048, ChanStat
    • MR — Send data to Rockland support  calibration text + .tbd/.sbd converted data (.dat ASCII) + logfile

    Depth ramp-up

    • Adjust dive depth and altitude incrementally  typical sequence: 100 → 300 → 600 → 990 m (edit yo35.ma)
    • Adjust autoballast values as appropriate
    • (optional) Toggle current correction in surface dialog  !put u_use_current_correction 0
    • Notify field crew that glider is on mission and diving to full depth

    4. Remainder of Day 1¶

    • Email BODC with deployment time and location
    • (optional) Increase number of half yos
    • Increase surface interval / no-comms count once navigation, data retrieval, and nominal operation are confirmed
    • Adjust engineering data density as desired  edit sbdlist.dat (restart mission)
    • Adjust science data density as desired  edit tbdlist.dat (send to science)
    • Compare actual results against campaign plan; verify with lead pilot that all requirements are being met
    • Subscribe to abort SMS/calls  GliderLab
    • Set SFMC email subscriptions  glider page → Options → Configure Glider Event Subscriptions
    • Adjust altimeter minimum depth  e.g. !put u_alt_min_depth 40 to enable altimeter at 40 m and deeper

    MicroRider (if fitted)

    • MR — Repeat MicroRider channel checks periodically (2–3× per deployment day)

    5. Day 2 and Beyond¶

    • Complete daily pilot log and post to C2  SLC-CL-PLT05 Pilot log checklist.xlsx
    • Confirm deployment-specific criteria are being satisfied: energy budget · science requirements · navigation requirements

    MicroRider (if fitted)

    • MR — Repeat probe capacitance check (MR1) once per week
    • MR — Repeat TBD data channel analysis (MR2) once per day
    • MR — Repeat mid-profile channel stats (MR3) when in doubt about probe health

    Sign-Off¶

    Phase Completed by Time (UTC) Notes
    Pre-Deployment
    On Deck
    In Water — first dive
    Depth ramp-up complete
